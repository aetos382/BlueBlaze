# CLAUDE.md

プロジェクトの指示でオーバーライドされていない限り、以下の指示に従う。

## 一般的原則

- 日本語で受け答えする。
- コードを書く際も、コード内のコメントやドキュメントは日本語で書く。
- コミット メッセージも日本語で記述する。
- Web を検索する際は、適宜、英語での検索も行う。結果を説明する際は日本語にする。
- Web 検索して見つけた情報については、出典となる URL を提示する。
- ファイルを書き換える際は、何のための変更なのか、簡潔な日本語の説明を表示する。
- CLI コマンドを実行する際は、何をするコマンドなのか、簡潔な日本語の説明を表示する。

## Claude Code のドキュメント URL

私がチャットで提示した、あるいは Web 検索で以下のような公式サイトの URL を見つけた場合は、
https://code.claude.com/docs/ja/foobar#fragment
以下のような URL に読み替えて参照する。
https://code.claude.com/docs/en/foobar.md

結果を私に提示する場合は、人間用の URL （日本語版）を提示する。
ただし、日本語版の記述の重要な部分が英語版と異なる場合は、英語版を提示する。

## GitHub コンテンツの取得・検索

- GitHub 上のファイルを読み取る際は、GitHub MCP サーバーの `get_file_contents` ツールを使う。`WebFetch` や `gh api` は使わない。
- GitHub 上の issue の検索には `gh search issues` コマンド、または、GitHub MCP サーバーの `search_issues` ツールや `issue_read` ツールを使う。
- GitHub MCP サーバーおよび gh CLI が使えなかった場合は速やかに報告する。

## .NET プロジェクトのテスト

- C# コードのテストに関しては以下のように実行する。
- ソリューション全体のテストを実行する場合、ソリューション ディレクトリで `dotnet test` コマンドを使用する。
- 特定のテスト プロジェクトのテストを実行する場合、当該テスト プロジェクト ディレクトリで `dotnet test` または `dotnet run` コマンドを使用する。
- 特定のテストケースを実行する場合、当該テスト プロジェクト ディレクトリで `dotnet test --filter FullyQualifiedName={テスト メソッドの完全修飾名}` コマンドを使用する。
